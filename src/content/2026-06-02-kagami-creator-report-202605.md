Kagami Creator Report (May 2026) 
บันทึกแนวคิดการทำคอนเทนต์เพจคางามิ (เดือนพฤษภาคม 2026)

การเขียนข่าวเกมวิชวลโนเวลเป็นงานที่ผมต้องทำในฐานะแอดมินเพจ ขั้นตอนคือ
1. อ่านโพสต์จาก Twitter
2. จัดหมวดหมู่ เช่น เปิดตัวใหม่ อัปเดตข้อมูล ประกาศมาสเตอร์อัพ วางจำหน่าย
3. นำข้อมูลที่เกี่ยวข้อง เช่น ผู้พัฒนา ชื่อเกม เว็บไซต์ มากรอกในแบบฟอร์มที่เตรียมไว้แล้ว
4a.  ตรวจสอบข้อมูลจากเว็บ VNDB
4b. ตรวจสอบข้อมูลจากร้านค้า Steam (ถ้ามี)
5. คัดลอกข้อความทั้งหมดไปโพสต์ในเพจ Kagami Visual Novel

ผมวิเคราะห์แล้วว่างานที่มีลักษณะเป็น routine และมีแบบแผนที่ชัดเจนแบบนี้สามารถเอาเอไอมาช่วยงานได้ เลยไม่รอช้า พัฒนาระบบช่วยเขียนข่าวเกมวิชวลโนเวลขึ้นมา

.........

พอแปะลิงก์ทวิตเตอร์ ระบบจะเขียนข่าวให้เสร็จพร้อมใช้งานใน 30 วินาที แต่ข้อผิดพลาดเยอะเกินกว่าที่นำมาใช้งานจริงได้ เช่น
- "ผู้พัฒนาเกมเดือนสิงหาคม" <-- August คือชื่อค่ายเกม ไม่ใช่เดือนสิงหาคม
- "ซากปรักหักพัง และการพบพาน และการผจญภัย" <-- จะแปลชื่อเฉพาะทำไม แล้วคำว่า Relilium ที่เป็นชื่อเกมหายไปไหน
- "เนื้อเรื่อง: พระเอกได้พบกับ osananajimi โดยบังเอิญ" <-- คำสามัญทั่วไปแบบนี้ดันไม่แปล

นอกจากนี้ยังมีปัญหาเชิงเทคนิคอีกเพียบ เช่น 
- ทวิตเตอร์ค่ายเกมบางค่ายจำกัดอายุไว้ 18+ ขึ้นไป ไม่ได้เปิดสาธารณะ ระบบของผมเลยเข้าไปดูดข้อมูลไม่ได้ ต้องเลี่ยงระบบโดยใช้ proxy แทน 
- สั่งให้ค้นหาข้อมูลในเว็บ VNDB แต่ระบบหาข้อมูลไม่เจอ ผมเลยเขียนคำสั่งให้ query ข้อมูลจาก API ของเว็บ VNDB แทน
- บางครั้งระบบจะไม่เชื่อฟัง instruction ที่เขียนไว้ ถือวิสาสะไปหาข้อมูลเอง เปลือง token สุดๆ
